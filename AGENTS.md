## 一、專案定位

本專案是**獨立的心理學文本分析系統**——不屬於 ECC/EME 兩院制蘇維埃體系。它從 ECC 兩年開發經驗中萃取方法論，應用於真實心理學理論文獻的結構化分析。

### 核心語料

| 子語料庫 | 冊數 | 時期 | 核心主題 |
|------|:---:|------|------|
| vygotsky | 7 | 1924–1934（+私人筆記至 1934） | 文化歷史心理學、高階心理功能、內在言語、ZPD |
| psychoanalysis | 5 | 1905–2014（核心 1905–1939） | 精神分析史、Spielrein、Freud/Jung 分裂、俄國精神分析 |

## 三、核心原則（Always-On Rules）

### 3.1 認識論公理

| # | 公理 | 含義 |
|---|------|------|
| 1 | **雙世界線不可混淆** | α₁（真實心理學史）和 γ（理論張力分析）是獨立的信號維度 |
| 2 | **可能性空間優先於單一實例** | 被邊緣化的替代理論方案構成「可能性空間」 |
| 3 | **制度重力與理論分叉動能** | 制度密度施加收斂重力；制度崩潰釋放分叉動能 |
| 4 | **殘差即種子** | 學派間不可通約的概念標誌理論分叉點，不是需要消除的差異 |
| 5 | **DCA 構成 UCD；T-SLAC 是觀察中介** | DCA 分支樹遞歸構成理論 UCD；tt 頻譜是觀察中介 |

### 3.2 操作規則

| 原則 | 說明 |
|------|------|
| **SSOT** | YAML Frontmatter 是文獻元數據的唯一真實來源。禁止創建與之競爭的中央數據庫。 |
| **寧缺勿濫** | 無法確定的日期、作者、概念標記→保留 `null`。嚴禁憑空發明。 |
| **來源不可觸動** | `corpus/` 下的原始文獻不可修改。所有分析產出存放在 `index/` 和 `analysis/`。 |
| **比較類型學** | 不同學派的概念不可直接「翻譯」或「融合」——比較必須標明類型（0/I/II/III/IV/V）。tt_vyg 和 tt_pa 原始值不可直接比較——僅通過百分位（類型 III）。 |
| **多尺度認知** | 所有語義判斷應攜帶頻譜坐標（長波/中波/短波相位）。禁止無視多尺度干涉的孤立分析。 |

## 四、邊界（Boundaries）

### 🚫 NEVER

| 操作 | 原因 |
|------|------|
| 修改 `corpus/` 下的原始文獻 | 來源不可觸動 |
| `git commit` / `git push` / `git reset` | 必須由人類執行 |
| 憑空發明日期、作者、概念標記 | 寧缺勿濫原則 |
| 將 API Key / Token 寫入會被 git 追蹤的檔案 | 安全紅線 |

### ⚠️ ASK FIRST

| 操作 | 原因 |
|------|------|
| 刪除 `system-template/scripts/` 下的文件 | 可能破壞基礎設施 |
| 引入新的 Python 依賴 | 需確認環境隔離 |
| 修改 `GRANDPLAN.md` 的 Phase Gate 狀態 | 需人類確認 |

### ✅ ALWAYS

| 操作 | 說明 |
|------|------|
| 結構變更後運行 `validate-schema.py` | 確保 Frontmatter 格式正確 |
| 修改 `index/documents-raw/` 後重建編譯庫 | `build-compiled-db.py` |
| 新文獻加入後更新對應 `INDEX.md` | 保持索引看板同步 |
